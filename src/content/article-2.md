## 二、TRAE 的三大杀手锏

### 🏆 杀手锏 1：AI 模式选择

TRAE 提供了多种 AI 交互模式，这是它和 Cursor 最大的区别：

**Agent 模式**：
- 自动分析项目结构
- 自主安装依赖、执行命令
- 端到端完成任务，无需手动干预

**Edit 模式**：
- 精确选中代码段进行修改
- 支持差异对比（Diff View）
- 适合精细调整已有代码

**Chat 模式**：
- 独立的对话窗口，不干扰编辑器
- 支持多轮追问和深入讨论
- 适合架构设计评审和技术方案讨论

> 这三种模式的灵活切换，让 TRAE 既能做"自动编程"也能做"辅助编程"——这是其他 IDE 不具备的。

### 🏆 杀手锏 2：中文指令理解能力

作为国产 AI IDE，TRAE 对中文的理解远超 Cursor 和 Copilot：

```javascript
// TRAE 能理解这样的中文需求：
// "给这个按钮加个点击动画，点一下旋转180度，然后再点回来"

function AnimatedButton({ children, onClick }) {
  const [rotated, setRotated] = useState(false);

  const handleClick = () => {
    setRotated(!rotated);
    onClick?.();
  };

  return (
    <button
      onClick={handleClick}
      className={`transition-transform duration-300 ${
        rotated ? 'rotate-180' : 'rotate-0'
      }`}
    >
      {children}
    </button>
  );
}
```

### 🏆 杀手锏 3：Agent 模式下的自动化能力

TRAE 的 Agent 模式是我用得最多的功能。它不仅能写代码，还能：

```
┌─ 项目启动 ──────────────────────────────┐
```

## 三、实际使用场景与技巧

### 场景 1：快速原型开发

需求：从零搭建一个量化回测结果可视化仪表盘

```bash
# 只需告诉 TRAE：
# "帮我创建一个 React 仪表盘组件，展示回测的收益率曲线、
#  最大回撤、夏普比率等指标，使用 Recharts 绘图"
```

TRAE 会自动：
1. 安装 `recharts` 依赖
2. 创建仪表盘组件
3. 添加模拟数据
4. 接入路由

整个过程 **5 分钟**，比手动写快 **10 倍**。

### 场景 2：Bug 修复

遇到 Bug 时的最佳实践：

```
1. 复现 Bug（截图或描述）
2. 在 TRAE Chat 中描述："这个页面在暗色模式下字体看不清"
3. TRAE 分析并定位到问题文件
4. 切换到 Edit 模式，精确修改 CSS
5. 预览效果，确认修复
```

### 场景 3：代码重构

```
"帮我把这个 class 组件改成函数式组件"
TRAE → 分析 → 重构 → 对比 diff → 应用修改
```

## 四、高效使用技巧

### 4.1 规则文件（.trae/rules）

TRAE 支持项目级规则配置。在 `.trae/rules/` 目录下创建规则文件：

```
.trae/rules/
├── project_rules.md    # 项目通用规则
├── frontend_rules.md   # 前端专用规则
└── backend_rules.md    # 后端专用规则
```

示例 `.trae/rules/project_rules.md`：

```markdown
# 项目规则

## 技术栈
- React 18 + Vite
- Tailwind CSS
- FastAPI + PostgreSQL

## 约束
- 不修改 package-lock.json
- 后端 API 返回统一格式: { code, data, message }
- 组件放在 src/components/ 下
```

### 4.2 用好 Context 上下文

TRAE 的 Agent 模式会自动分析项目上下文，但有时需要手动添加：

```
@src/utils/api.js   # 引用特定文件让 AI 理解
```

### 4.3 合理拆分任务

不要一次性让 TRAE 做太多事，推荐拆分为：

```
✅ "帮我创建用户登录表单组件"
→ 完成后
✅ "给表单添加表单验证逻辑"
→ 完成后
✅ "接入登录 API"
```

而不是：

```
❌ "帮我做一个完整的用户系统，包括登录注册密码重置..."
```

### 4.4 SOLO 模式——专注工作目录

SOLO 模式是 TRAE 中非常实用的**目录专注功能**。使用前先定位到具体工作目录，能让 AI 把注意力 **Focus** 在该目录下，避免跑偏到项目中其他无关部分。

```bash
# 使用方式
# 1. 在 TRAE 中打开目标项目目录
# 2. 切换到 SOLO 模式
# 3. AI 的操作范围自动限定在当前工作目录
```

**适用场景**：
| 场景 | 说明 |
|------|------|
| **Monorepo 项目** | 多个子项目共存，只想专注修改其中一个 |
| **大型代码库** | 代码量庞大，避免 AI 扫描不相关模块浪费时间 |
| **跨项目开发** | 同时打开多个项目窗口，每个窗口各自 Focus |

> 💡 **最佳实践**：在 SOLO 模式下，TRAE 的上下文感知仅限定在工作目录内，既节省 Token 又提升生成准确度。

### 4.5 自定义 Agents——实现特定功能

TRAE 支持创建**自定义 Agent**，为特定任务预设行为和约束：

```bash
# 创建自定义 Agent 的步骤
# 1. 点击 Agent 下拉菜单 → 管理 Agents
# 2. 新建 Agent，配置以下内容
# 3. 保存后即可在 Agent 选择器中切换
```

**自定义 Agent 配置示例**：

```
Agent 名称:         前端开发助手
适用技术栈:         React + Tailwind CSS + TypeScript
行为规则:
- 优先使用函数式组件 + Hooks
- 组件文件放在 src/components/ 下
- 样式统一使用 Tailwind 类名
- 自动添加 PropTypes 或类型定义
```

```
Agent 名称:         API 开发助手
适用技术栈:         FastAPI + PostgreSQL
行为规则:
- 遵循 RESTful 规范
- 返回统一格式 { code, data, message }
- 自动添加输入验证和错误处理
- 每个接口写文档注释
```

**管理员宝箱示例**：

```
Agent 名称:         管理员宝箱
功能:               每日服务器巡检
行为规则:
- 检查 CPU / 内存 / 磁盘使用率
- 分析 Nginx 错误日志
- 检查 SSL 证书过期时间
- 生成巡检报告 Markdown
- 默认使用 SOLO 模式连接服务器目录
```

> 💡 **建议**：为不同角色创建专属 Agent（前端/后端/DevOps），切换任务时直接选用，无需重复描述上下文。

### 4.6 模型选择建议

TRAE 支持多种 AI 模型，不同场景选择不同模型能兼顾**速度**与**质量**：

| 模型 | 推荐场景 | 响应速度 | 准确度 | 备注 |
|------|---------|:--------:|:------:|------|
| **DeepSeek V4 Flash** | 日常编码、快速修改 | ⚡⚡⚡⚡⚡ 飞速 | ✅ 较高 | **免费首选**，绝大部分场景够用 |
| **GLM V5 Turbo** | 多模态任务（图片理解、UI 截图生代码） | ⚡⚡⚡ 快 | ✅ 高 | 支持图像输入，适合前端切图 |
| **DeepSeek V4 Pro** | 复杂算法、架构设计 | 🐢 需排队 | 🌟🌟🌟 极高 | 上午较空闲，排队时间短 |
| **GLM 5.1** | 高难度任务（安全审计、大规模重构） | 🐢 需排队 | 🌟🌟🌟 极高 | 建议安排在上半天使用 |

```bash
# 模型切换建议
# 日常编码 → DeepSeek V4 Flash（免费、飞速、够用）
# 多模态分析 → GLM V5 Turbo（截图生成代码）
# 复杂任务 → DeepSeek V4 Pro / GLM 5.1（上午使用，排队更少）
```

> 💡 **经验之谈**：DeepSeek V4 Flash 的响应速度极快，准确度也相当不错，完全可以作为**主力模型**应对 80% 的日常任务。高难度任务再切换到 Pro 或 GLM 5.1，并尽量安排在**上午时段**以避开排队高峰。

### 4.7 多任务并行

TRAE 支持**多个任务同时进行**，通过左上角的「新建任务」按钮可以创建独立的对话 Tab：

```bash
# 多任务管理技巧
# 任务 A: 重构用户模块（Agent 模式，后台持续运行）
# 任务 B: 写 API 文档（Chat 模式，边写边问）
# 任务 C: 修复某个 Bug（Edit 模式，快速定位修改）
```

**为什么推荐多任务并行**：
- 等待 Agent 执行长任务时，切到另一个任务继续工作，互不阻塞
- 不同任务使用不同模式（Agent / Edit / Chat），互不干扰
- 每个任务独立维护上下文，不混淆

> 💡 **建议**：将大型任务拆分为多个并行子任务，配合自定义 Agent，能大幅缩短总开发时间。

### 4.8 提示词优化

在发送任务给 AI **之前**，先点击输入框旁的「提示词优化」按钮！这个小功能经常被忽略，但效果非常显著：

```bash
# 原始 Prompt:
"帮我写个登录页面"

# 优化后的 Prompt:
"创建一个 React 登录表单组件，包含用户名和密码输入框、
 登录按钮、表单验证（非空检查、密码长度≥6位）、
 错误提示信息显示区域，使用 Tailwind CSS 样式。"
```

| 操作 | 效果 |
|------|------|
| ❌ 直接发送模糊指令 | AI 自由发挥，结果可能偏离预期 |
| ✅ **点击提示词优化** | AI 自动补充细节、明确约束、规范格式 |
| ✅ 优化后再检查调整 | 准确度提升 **30%-50%** |

> 💡 **工作流**：写需求 → 点「提示词优化」→ 微调确认 → 发送给 AI，养成这个习惯后代码生成质量会有质的飞跃。

### 4.9 Remote 插件——SSH 连接与 SFTP

TRAE 的 **Remote 插件**支持通过 SSH 连接远程服务器，直接在 IDE 中管理远程文件：

```bash
# 安装 Remote 插件
# 1. 在插件市场搜索 "Remote" 并安装
# 2. 点击左侧 Remote 图标
# 3. 添加 SSH 连接：ssh user@hostname -p port
# 4. 连接成功后即可浏览远程目录
```

**核心功能**：

| 功能 | 说明 |
|------|------|
| **SSH 终端** | 直接在 TRAE 中打开远程 Shell |
| **目录文件管理** | 像操作本地文件一样浏览、编辑远程文件 |
| **SFTP 传输** | 拖拽上传/下载文件，非常流畅 |
| **远程工作区** | 将远程目录作为项目打开，支持完整的 AI 功能 |

> 💡 **实用场景**：开发完成后通过 Remote 插件连接服务器，直接在远程目录上用 TRAE 的 Agent 模式做调试和配置修改。SFTP 传输文件比传统 FTP 客户端方便得多。
